Version control practical

Form into pairs.

One of you offer to share your repository:

  • Get your partner to give you their BitBucket username.
  • In BitBucket, click on the cog, and click Access Management.
  • Enter your partner's username, select Write and click Add.

Now both of you change into your home directory and clone the repository, using a command like the following and replacing USERNAME and OWNERUSERNAME with your username and the username of the repository you're sharing: 

cd
hg clone https://USERNAME@bitbucket.org/OWNERUSERNAME/software-carpentry-boot-camp mycopy
cd mycopy

Now both try editing the same file and commiting the changes.

Now both try to push: 

hg push https://USERNAME@bitbucket.org/OWNERUSERNAME/software-carpentry-boot-camp

If you get a "Push failed" then you need to:

  • Pull down the latest copy
  • Merge the changes
  • Resolve the conflicts
  • Mark conflicts as resolved, using 
    hg resolve -m FILENAME
  • Commit the updated files
  • Push these
